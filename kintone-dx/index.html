current problems

現在の課題

📨
依頼・質問が埋もれる

Chatworkのやり取りに紛れて対応漏れが発生。ステータス管理ができない。

👤
属人化している

個人が案件を管理。チーム全体で進捗が見えない。担当不在時に対応できない。

📝
議事録作成が手間

訪問・Web会議の議事録を毎回手作業で作成。共有にも時間がかかる。

🔄
二重入力が多い

PCAの請求・給与データをExcelに手動で転記。ミスのリスクも。

🔍
横断確認ができない

複数の顧問先の依頼を一覧で確認できず、優先順位が見えない。

📊
Excel管理の限界

月次処理の確認作業がExcel頼み。データ活用や自動化が困難。

solutions

課題と解決策

#現状の課題解決策
1依頼・質問がChatworkに埋もれるKintoneスペースで依頼・質問を別アプリ化してステータス管理
2対応漏れが発生するステータス・担当者管理で「放置」を可視化
3複数顧問先の依頼を横断確認できない社内Kintoneに全顧問先の依頼・質問を自動集約
4議事録作成・共有に手間がかかるGoogle Meet → GASで自動要約 → Kintone記録＆Chatwork通知
5PCAデータを手動で転記しているPCAクラウドAPIでKintoneに自動連携
6訪問履歴がバラバラ履歴アプリに一元管理

communication

スペース運用（Chatwork併用）

Chatworkを廃止せず、役割を分担して併用します

Chatwork（継続利用）
  • 💬顧問先への通知・連絡（従来通り）
  • 📢Kintone更新時のGAS自動通知先
  • 👤顧問先は今まで通りの使い心地
Kintoneスペース（新設）
  • 📥依頼の受付・ステータス管理
  • ❓質問の受付・ステータス管理
  • 📢情報提供（法改正・お知らせ）
  • 📦社内DBへ自動複写で横断管理
📂 顧問先別スペース（例：○○社スペース） ├── 💬 チャット（スレッド毎）　← 顧問先と双方向 ├── 📥 依頼アプリ　← 区分・ステータス・担当者管理 │　　└── 🟣 複写 → 社内「依頼DB」 ├── ❓ 質問アプリ　← 区分・ステータス・担当者管理 │　　└── 🟣 複写 → 社内「質問DB」 └── 📢 情報提供アプリ　← 事務所→顧問先への発信
automation

GAS 自動化エンジン

📹 Google Meet → 自動議事録
🎙️ Google Meetで会議
↓ Googleが自動文字起こし
↓ GASが要約を自動作成
↓ Kintone「履歴」に記録 + Chatwork通知
✨ 議事録作成の工数をほぼゼロに
📊 PCAクラウド → 自動連携
📦 PCAクラウド商魂（請求書）
↓ API → Kintone「案件」
👥 PCAクラウド給与
↓ API → Kintone「従業員」
✨ 二重入力の排除・データ一元管理
💬 Kintone → Chatwork通知
📋 Kintoneのレコード更新
↓ GASが変更内容を整形
↓ Chatworkで顧問先に自動通知
✨ リアルタイムで状況共有
