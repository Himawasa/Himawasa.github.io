4. 3社の強み・弱み比較

GCP（Google Cloud Platform）

💪 強み⚡ 弱み
✅ AI / ML が圧倒的に強い（Vertex AI, Gemini）❌ AWSに比べ情報量・コミュニティが小さい
✅ コンテナ技術のリーダー（Kubernetes開発元）❌ エンタープライズ実績ではAWS/Azureに劣る
✅ Google Workspace と同一アカウント管理❌ サービス数がAWSほど多くない
✅ 無料枠が充実（$300クレジット、Cloud Run無料枠）❌ サポートプランが高め
✅ UIが直感的で学習コストが低い
✅ 固定IP（NAT）が圧倒的に安い

AWS（Amazon Web Services）

💪 強み⚡ 弱み
✅ 市場シェアNo.1（業界標準）❌ NAT Gatewayが非常に高い（$0.062/h）
✅ サービス数が最多（200以上）❌ 料金体系が複雑で把握が困難
✅ 技術情報・事例が最も豊富❌ 0スケール非対応（Fargate）で小規模に不向き
✅ エンジニア採用がしやすい❌ Google Workspace連携なし
✅ 大規模ワークロードに最適❌ 初回無料クレジットなし

Azure（Microsoft Azure）

💪 強み⚡ 弱み
✅ Microsoft 365とシームレス連携❌ NAT Gatewayが高い（$0.045/h）
✅ Azure OpenAI Service（GPT系AIを法人利用可能）❌ MS製品以外との相性がやや劣る
✅ エンタープライズ向け機能が充実❌ UIの操作性がGCPに比べ複雑
✅ Container Appsは0スケール対応❌ 日本語の技術情報がAWSより少ない
✅ $200無料クレジット（初回）❌ GWSアドオン連携は不可

もしAzureを選ぶなら？

💡 Azureが適するケース Microsoft 365を全社で利用していて、Active DirectoryやTeamsとの連携が最優先の場合。
今回のケースではGoogle Workspaceを契約するため、GCPとの親和性が圧倒的に高い。

もしAWSを選ぶなら？

⚠️ AWSが適するケース 大規模トラフィック・複数リージョン展開・豊富なAWSエンジニアが社内にいる場合。
今回のような小規模1拠点のポータルには、コスト・機能ともにオーバースペック。
