## 5. トークン管理API

### 有効確認（最初に叩いて確認する）
```
GET /tokens/{token}/available
```
レスポンス例：
```json
{ "available": true }
```

### 再発行（トークンが古い場合）
```
POST /tokens/{token}
```
新しいトークンが返ってくる。

### 利用停止
```
DELETE /tokens/{token}
```

## 6. 今回使うAPI（シフト表自動出力に必要なもの）

### ① 従業員データ取得
```
GET /employees
```
**用途**：氏名・所属・雇用区分を取得して、シフト表の職員欄を埋める

**オプションパラメータ**：
| パラメータ | 説明 | 例 |
|-----------|------|----|
| date | 指定日時点の在籍者 | `2026-04-01` |
| division | 所属コードで絞り込み | `1000` |
| includeResigner | 退職者を含む | `true` |

**レスポンス例**：
```json
{
  "divisionCode": "1000",
  "divisionName": "本社",
  "typeCode": "1",
  "typeName": "正社員",
  "code": "1000",
  "key": "8b6ee646...",
  "lastName": "田中",
  "firstName": "裕子"
}
```

---

### ② 日別スケジュールデータ取得 ★メイン★
```
GET /daily-schedules/?start=2026-04-01&end=2026-04-30
```
**用途**：各職員の日別シフト（出勤・退勤予定時間）を取得

**オプションパラメータ**：
| パラメータ | 説明 | 例 |
|-----------|------|----|
| start | 期間開始日 | `2026-04-01` |
| end | 期間終了日（最大62日） | `2026-04-30` |
| division | 所属コードで絞り込み | `1000` |

**レスポンス例**：
```json
[
  {
    "date": "2026-04-01",
    "dailySchedules": [
      {
        "date": "2026-04-01",
        "employeeKey": "8b6ee646...",
        "clockInSchedule": "2026-04-01T09:00+09:00",
        "clockOutSchedule": "2026-04-01T18:00+09:00",
        "breakSchedule": 60,
        "workDayTypeName": "平日",
        "holidaysObtained": { ... }
      }
    ]
  }
]
```

**重要フィールド**：
| フィールド | 意味 |
|-----------|------|
| `clockInSchedule` | 出勤予定時間（例：9:00） |
| `clockOutSchedule` | 退勤予定時間（例：18:00） |
| `breakSchedule` | 休憩予定時間（分） |
| `workDayTypeName` | 勤務日種別（平日/法定休日/法定外休日） |
| `holidaysObtained.fulltimeHoliday` | 全日休暇情報 |

---

### ③ 特定日のスケジュール取得
```
GET /daily-schedules/2026-04-01
```
指定日1日分だけ取得する場合。

## 7. シフト表自動出力の処理フロー

```
【毎月1回実行（月末〜翌月初）】

① GET /employees
   └─ 職員リスト取得（氏名・資格・雇用区分）
   
② GET /daily-schedules/?start=YYYY-MM-01&end=YYYY-MM-31
   └─ 月全体のシフト取得
   
③ 時刻→シフト記号変換
   └─ 例: 9:00〜18:00 = "a"
   └─ 例: 8:30〜17:30 = "b"
   └─ ※シフト記号表はExcelのシフト記号表シートから読み取る
   
④ Excelテンプレートに転記
   └─ 行：職員（職種・氏名）
   └─ 列：日付（1〜31日）
   └─ セル：シフト記号
   
⑤ GoogleドライブにExcelで保存
   or PDFで出力
```

---

## 8. 注意点・課題

### シフト記号のマッピング（要確認）
KingOfTimeは「始業時刻・終業時刻」でシフトを管理している。  
行政提出Excelは「記号（a, b, c...）」で管理している。  
→ **時刻→記号の変換テーブルを作成する必要がある**

例：
| KingOfTime（時刻） | 行政Excelの記号 |
|-------------------|----------------|
| 9:00〜18:00 | a |
| 8:30〜17:30 | b |
| 早番（7:00〜16:00） | c |
| 夜勤（17:00〜翌9:00） | d |
| 休み | - |

> ⚠️ この対応表は施設ごとに異なるため、**初期設定時に施設担当者と確認が必要**

### GAS（Google Apps Script）での実装時の注意
- Bearerトークンは **PropertiesService** に保存すること（コード内に直書き禁止）
- IPアドレス制限がある場合、GASの固定IPは利用できない
  → GASからはIP制限なし（空欄）にしておく必要がある
  → または **許可IPを削除**してGASから叩く
